# 社交回声 · 中保真原型设计说明

> Social Echo — 终结"改天请你吃饭"的私人社交助理
> 中保真版本（Vue 3 + localStorage） · v0.1 · 课堂展示用

---

## 一、关键体验流程覆盖

中保真原型已覆盖 App 的 **7 个核心流程**：

| # | 流程节点 | 路由 | 解决的用户故事 |
| --- | --- | --- | --- |
| 1 | 首次引导 + 隐私承诺 | `/onboard` | "我担心隐私，凭什么把通讯录交给你？" |
| 2 | 价值速览首页 | `/` | "这 App 到底是干嘛的？" |
| 3 | 联系人卡片墙（温度可视化） | `/echo` | "我有谁好久没联系了？" |
| 4 | 联系人详情 + 破冰文案投喂 | `/echo/contact/:id` | "我要怎么开口才不尴尬？" |
| 5 | 通知中心（结冰预警 + 最近建议） | `/notifications` | "我又错过了哪些重要的人？" |
| 6 | 模拟朋友圈（内容触发器） | `/social` | "看到 ta 发了动态，但是不知道说啥" |
| 7 | 能量报告（社交复盘） | `/report` | "我这一周的社交健康吗？" |
| 8 | 设置 + 隐私偏好 | `/settings` | "我能不能调成我自己的节奏？" |

辅助闭环：
- **体面放弃** ：忽略 ≥ 设定阈值（默认 3）→ 卡片自动沉淀，可在卡片墙底部"沉淀区"折叠查看
- **复盘**：每条互动写入 `history`，详情页与报告页可回溯
- **重置**：设置页"清除全部本地数据"

## 二、原始草图 → 修改后原型：三处最大更改

### 更改 1：破冰文案 —— 从「单条结果 + 复制」 → 「三语气并列 + 触发理由可编辑」

**原始草图：**
```
┌─────────────────────────┐
│  AI 给你的开口理由：       │
│  "好久不见，最近怎么样？"  │
│                         │
│              [ 复制 ]    │
└─────────────────────────┘
```

**修改后：**
```
┌─ 触发理由 ────────────────────┐  ← 用户可编辑
│ ta 最近发了："新论文 accepted"  │
└────────────────────────────┘

┌─ 尊敬 (respect) ──────────┐
│ "周老师好，刚看到您新论文…" │ [ 用此条 ]
└────────────────────────┘
┌─ 平和 (neutral) ──────────┐
│ "周老师！恭喜！还记得当年…" │ [ 用此条 ]
└────────────────────────┘
┌─ 随意 (casual) ──────────┐
│ "卧槽老师太牛了哈哈，最近…" │ [ 用此条 ]
└────────────────────────┘

[ ↻ 重新生成 ]   [ × 都不喜欢 ]
```

**为什么这样改 → 可用性提升：**

1. **"匹配感"提升 ≈ 3 倍**：用户研究中「林小棠（纠结型）」明确表示，"AI 给的话听起来不像我会说的"。提供 3 种语气=3 倍的"听起来像我"的概率，避免一次没命中就放弃。
2. **触发理由透明化** ：把"为什么现在要联系 ta"显式展示并允许修改，符合 Nielsen 启发式 #1（系统状态可见）和 #3（用户控制）。用户不再是黑盒消费者。
3. **降低决策成本**：三选一比"接受/拒绝单选"心理负担更低 —— 用户的注意力从"AI 这个建议好不好"转移到"哪种语气更适合我现在的心情"，决策路径更短。
4. **明确的"都不喜欢"出口** ：把忽略也变成一种正向操作（贡献到「体面放弃」计数），而不是无声关闭页面，提升系统反馈完整度。

---

### 更改 2：结冰预警 —— 从「打开主产品才能看到」 → 「全局通知中心 + 红点徽章」

**原始草图：**
> 用户必须主动打开 App → 进入主产品 → 滚动卡片墙才能发现"咦谁结冰了"

**修改后：**
- 顶部导航栏新增「通知」入口，带 **红色数字徽章**（如 `通知 ③`），实时显示当前结冰联系人数量
- 独立 `/notifications` 页面，按"结冰天数倒序"排列，每条附 **触发建议** （"ta 最近发了…"）
- 折叠展开的「触发器说明」 让用户理解系统会基于哪些事件提醒（时间/资讯/动态/环境）

**为什么这样改 → 可用性提升：**

1. **从"主动检索"变成"被动感知"** ：原型测试时发现，用户进入卡片墙后会被卡片视觉吸引，反而忽略了底部已结冰的人。徽章把"重要但不紧急"的信息提到全局视野，符合 Krug《Don't Make Me Think》的"少思考"原则。
2. **可达性升级** ：所有页面（朋友圈/报告/设置）都能一眼看到当前有几个人需要处理，从 4 跳变成 1 跳。
3. **触发理由前置** ：在通知列表里就预先显示"ta 最近发了什么"，用户在点进详情前已经有了"开口的钩子"，减少冷启动空白页焦虑。
4. **避免推送疲劳** ：徽章是"软提醒"，不是手机系统推送，符合用研中"佛系陈默"对"无打扰"的诉求 —— 你想看就看，不想看就不看，但 App 不会替你决定何时打扰你。

---

### 更改 3：放弃机制 —— 从「主动删除联系人」 → 「3 次忽略自动沉淀 + 可恢复」

**原始草图：**
```
[ 长按联系人 ] → 弹出 [ 删除 ] 按钮 → 确认 → 永久消失
```

**修改后：**
- 用户每次"忽略"破冰建议，`ignoreCount + 1`
- 达到设置中的阈值（默认 3）→ 卡片自动从主列表移到 **「沉淀区」**（折叠在卡片墙底部，灰度+虚线边框）
- 沉淀区联系人 **不会被删除** ，仍可点击进入详情，任何一次主动互动（点赞、回复、生成文案）会自动恢复
- 设置页可调整阈值（1~10）

**为什么这样改 → 可用性提升：**

1. **降低"判死刑"心理负担** ：用研显示"赵一航（职场新人）"对"删除一个有过两面之缘的客户"有强烈犹豫——怕未来需要、怕显得绝情。"自然沉淀"提供了体面的中间状态：既不打扰当下，又保留可能性。
2. **行为驱动而非情绪驱动** ：让"放弃"由用户的实际行为（连续忽略）触发，而不是某个情绪化瞬间的"删除"决策，避免后悔操作。
3. **可逆设计** ：符合 Nielsen 启发式 #5（错误预防）和 #9（错误恢复）。任何主动互动都能让 ta 回来，无须从回收站找。
4. **承认社交节律** ：很多关系本来就是"阶段性高频"——同学时期、项目期、同城期。把"不再频繁联系"框定成 **客观状态**，而不是 **道德失败**，是产品对用户最大的善意。这一点也是项目最初的核心命题。

## 三、附：UI 风格说明（中保真特征）

| 特征 | 做法 |
| --- | --- |
| 视觉灰阶 | 主色 `#f5f5f5` 背景，`#888` 文字，唯一暖色仅出现在温度条与红点 |
| 线框感 | 大量 `1px dashed #c8c8c8` 占位框，`[ ... ]` 文字标识占位 |
| 结冰头像 | `filter: grayscale(1) brightness(0.85)` + `❄` 角标 |
| 不做精修 | 故意保留方角、单字号、无阴影，提示这是讨论稿而非视觉稿 |

> 中保真定位：**保留全部交互流程与信息架构**，刻意省略视觉精修，引导评审聚焦于"用户路径是否合理"而非"按钮颜色好不好看"。

## 四、与高保真版本的差异

| 维度 | 中保真（本版本） | 高保真（Next.js 版） |
| --- | --- | --- |
| 框架 | Vue 3 纯前端 | Next.js 14 App Router 全栈 |
| 数据 | localStorage | SQLite + Prisma |
| 文案生成 | 本地模板（`utils.js`） | DeepSeek 真实 API 调用 |
| 视觉 | 灰阶线框 | Tailwind 配色 + 渐变 |
| 用途 | **流程评审 / 可用性测试** | **真实功能验证 / 投资人 demo** |
